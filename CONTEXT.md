# Agent Readiness Skills

Reusable skills for preparing repositories and tasks for coding agents without turning procedures into static project documentation.

## Language

**Skill**:
A reusable agent procedure packaged as `SKILL.md` plus optional references or scripts.
_Avoid_: standard, manual, checklist pack

**Procedure**:
A repeatable way of working that belongs in a skill.
_Avoid_: project fact, policy

**Project fact**:
Information true for one repository: commands, owners, forbidden areas, tool policy, domain terms, and ADRs.
_Avoid_: generic agent advice

**Agent docs**:
Small project-owned files that expose project facts to agents.
_Avoid_: readiness pack, context dump

**Task context**:
The minimal evidence needed for one task: scope, relevant files, constraints, and verification.
_Avoid_: repository dump

**Agent brief**:
A durable implementation contract for an agent, usually attached to an issue or task.
_Avoid_: prompt, ticket dump

**Runtime policy**:
Rules for which tools, connectors, skills, hooks, side effects, and approvals are allowed.
_Avoid_: personal IDE settings
